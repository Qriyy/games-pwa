# 🏢 公司组织架构 — 棋牌乐园

## 组织架构图

```
                        👑 CEO（你）
                     定方向 · 接客户 · 验收
                            │
                        🧑‍💼 主管（我）
                调度 · 审核 · 部署 · 代码审查
                            │
          ┌─────────────────┼─────────────────┐
          │                 │                 │
      🎮 策划部          💻 技术部          🧪 测试部
     ┌────┴────┐      ┌────┼────┐       ┌────┴────┐
     │         │      │    │    │       │         │
  📝策划    🎨设计  🖥️前端 🤖AI 🔊音效  🔍功能   📱兼容
  规则/数值  UI/视觉  开发  算法  动效   测试     测试
```

---

## 各岗位详细职责

### 👑 CEO（你）
- **职责**：定方向、接客户需求、最终验收
- **日常**：「做XX游戏」「行/不行」「这里要改」
- **不做**：技术、设计、测试的任何事情

---

### 🧑‍💼 主管（Hermes — 我）
- **职责**：全公司调度管理
- **日常**：
  - 接CEO需求 → 拆解任务 → 分配给各部门
  - 代码审查（技术部产出的代码我来review）
  - 代码部署（git push → GitHub Pages）
  - 协调各部门进度
- **不做**：不亲自写代码、不亲自做设计

---

### 🎮 策划部

#### 📝 游戏策划（Claude Code A-1）
| 项目 | 说明 |
|------|------|
| 核心职责 | 游戏规则设计、玩法设计 |
| 产出物 | 策划文档（规则、流程、特殊玩法）|

#### 🔢 数值策划（Claude Code A-2）
| 项目 | 说明 |
|------|------|

| 例子 | 自摸×3、点炮×1、AI简单/中等/困难的决策逻辑 |

#### 🎨 UI设计（Claude Code A-3）
| 项目 | 说明 |
|------|------|
| 核心职责 | 界面布局、视觉风格、交互方案 |
| 产出物 | 设计稿（布局图、配色方案、交互说明）|
| 例子 | 棋盘样式、按钮位置、动画效果方向 |

## 开发流水线

```
① 👑 CEO → 「做五子棋」
           ↓
② 🎮 策划部 → 策划文档 + 数值方案 + UI方案（并行）
           ↓
③ 🧑‍💼 主管 → 审核策划案，通过则进入开发
           ↓
④ 💻 技术部 → 前端开发 + AI实现 + 音效制作（并行）
           ↓
⑤ 🧑‍💼 主管 → 代码审查
           ↓
⑥ 🧪 测试部 → 功能测试 + 兼容测试（并行）
           ↓
⑦ 🧑‍💼 主管 → 审核测试报告，有Bug则打回④
           ↓
⑧ 🧑‍💼 主管 → 部署上线（GitHub Pages）
           ↓
⑨ 👑 CEO → 验收
```
