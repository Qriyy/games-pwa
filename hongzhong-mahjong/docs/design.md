## 1. 游戏概述

| 项目 | 说明 |
|------|------|
| 游戏名称 | 红中麻将 |
| 玩家人数 | **4 人**（1 真人 + 3 AI，或 4 真人联网） |
| 牌组 | 万、条、筒各 1-9 × 4 张 + 红中 × 4 张，共 **112 张** |
| 核心特色 | **红中为万能牌（癞子）**，可替代任意一张牌完成胡牌 |
| 游戏目标 | 最先将手牌组成合法胡牌牌型的玩家获胜 |
| 单局时长 | 约 3-8 分钟 |

### 1.1 术语表

| 术语 | 含义 |
|------|------|
| 面子 | 顺子或刻子的统称 |
| 顺子 | 同花色连续三张牌，如一万二万三万 |
| 刻子 | 三张相同的牌，如三条三条三条 |
| 将（雀头） | 两张相同的牌，作为胡牌的基础对子 |
| 癞子 / 万能牌 | 红中，可替代任意一张牌 |
| 自摸 | 自己从牌墙摸到所需牌而胡牌 |
| 点炮（放炮） | 打出的牌被其他玩家胡牌 |
| 吃 | 用上家打出的牌与自己手中两张组成顺子（本规则中 **不允许吃牌**） |
| 碰 | 任意玩家打出的牌与自己手中两张相同，可碰 |
| 杠 | 四张相同的牌，分明杠和暗杠 |

## 2. 牌型定义

### 2.1 牌组构成

```
万子牌：一万、二万、三万、四万、五万、六万、七万、八万、九万  × 4 套 = 36 张
条子牌：一条、二条、三条、四条、五条、六条、七条、八条、九条  × 4 套 = 36 张
筒子牌：一筒、二筒、三筒、四筒、五筒、六筒、七筒、八筒、九筒  × 4 套 = 36 张
字  牌：红中                                                    × 4 张 =  4 张
─────────────────────────────────────────────────────────────────────────────
合计：112 张
```

### 2.2 牌面编码方案

开发时统一使用数值编码：

```typescript
// 花色编码：万=0x10, 条=0x20, 筒=0x30, 字=0x40
// 牌面值 = 花色 + 点数（1-9）
// 红中 = 0x41

enum Suit { WAN = 0x10, TIAO = 0x20, TONG = 0x30, ZI = 0x40 }

// 生成完整牌组
function generateDeck(): number[] {
  const deck: number[] = [];
  for (const suit of [Suit.WAN, Suit.TIAO, Suit.TONG]) {
    for (let rank = 1; rank <= 9; rank++) {
      for (let copy = 0; copy < 4; copy++) {
        deck.push(suit + rank);
      }
    }
  }
  // 红中 ×4
  for (let i = 0; i < 4; i++) deck.push(0x41);
  return deck; // length = 112
}
```

### 2.3 牌面显示

| 编码 | 显示 | Unicode |
|------|------|---------|
| 0x11 ~ 0x19 | 一万 ~ 九万 | 🀇 ~ 🀏 |
| 0x21 ~ 0x29 | 一条 ~ 九条 | 🀐 ~ 🀘 |
| 0x31 ~ 0x39 | 一筒 ~ 九筒 | 🀙 ~ 🀡 |
| 0x41 | 红中 | 🀄 |

> **开发提示**：实际项目中使用自绘 SVG/PNG 牌面素材，Unicode 仅作调试参考。

## 3. 游戏流程

### 3.1 总体流程图

```
┌──────────┐
│  游戏开始  │
└─────┬────┘
      ▼
┌──────────┐
│ 洗牌 & 切牌 │  ← Fisher-Yates 洗牌算法
└─────┬────┘
      ▼
┌──────────┐
│  庄家确定   │  ← 首局随机；之后轮流坐庄（胡牌者坐庄）
└─────┬────┘
      ▼
┌──────────┐
│  发  牌    │  ← 每人 13 张，庄家多摸 1 张（共 14 张）
└─────┬────┘
      ▼
┌──────────────────────────────────────────────┐
│                  游戏主循环                     │
│  ┌────────┐    ┌────────┐    ┌────────────┐   │
│  │ 摸 1 张  │───→│ 出 1 张  │───→│ 判定：碰/杠/胡 │   │
│  └────────┘    └────────┘    └─────┬──────┘   │
│       ▲                            │           │
│       └────────────────────────────┘           │
│                    (无人胡/碰/杠时下家摸牌)       │
└──────────────────────┬───────────────────────┘
                       ▼
               ┌──────────────┐
               │  胡牌 / 流局   │
               └──────┬───────┘
                      ▼
               ┌──────────────┐
               │   结算 & 结束   │
               └──────────────┘
```

### 3.2 详细流程

#### 3.2.1 洗牌

```
1. 将 112 张牌放入牌池数组
2. 使用 Fisher-Yates 算法随机打乱
3. 洗牌完成，牌池设为 face-down 状态
```

#### 3.2.2 发牌

```
1. 从牌池顶部（数组末尾）开始发牌
2. 按座次顺序（东→南→西→北），每人发 13 张（逐张发放或一次性发放均可）
3. 庄家额外摸 1 张，手牌共 14 张
4. 剩余牌组成"牌墙"（drawPile），从尾部依次摸取
```

> **牌墙数量**：112 - 13×3 - 14 = **59 张**（含海底牌）

#### 3.2.3 摸牌阶段

- 轮到当前玩家时，从牌墙顶部摸 1 张
- 庄家首回合跳过摸牌（手中已有 14 张）

#### 3.2.4 出牌阶段

- 摸牌后，玩家必须打出 1 张牌到弃牌堆
- 出牌前系统自动检测是否可以 **自摸胡牌** 或 **暗杠**
- 出牌后系统检测其他玩家是否可以 **碰 / 明杠 / 点炮胡**

#### 3.2.5 响应优先级

当一张牌被打出后，可能有多名玩家同时响应，按以下优先级判定：

```
胡（最高） > 杠 > 碰 > 过（最低）
```

- 同级响应按出牌者的下家 → 再下家 → 对家顺序优先
- 玩家有 10 秒响应时间，超时自动"过"

#### 3.2.6 流局判定

当牌墙剩余牌数 ≤ **14 张**（即保留 14 张不可摸的"海底"）时：

```
1. 停止摸牌
2. 本局判定为流局（荒牌）
3. 所有玩家不计输赢，庄家轮转
```

## 4. 胡牌规则

### 4.1 基本胡牌牌型

胡牌条件：**手牌（含刚摸到的牌） = 4 个面子 + 1 个将（雀头）**

共 14 张牌，结构如下：

```
[面子1] + [面子2] + [面子3] + [面子4] + [将对]
  3张       3张       3张       3张       2张    = 14 张
```

其中：
- **面子** = 顺子（同花色连续三张）或 刻子（三张相同）
- **将** = 两张相同的牌

### 4.2 面子类型示例

| 类型 | 示例 | 说明 |
|------|------|------|
| 顺子 | 一万 二万 三万 | 同花色连续三张 |
| 顺子 | 七条 八条 九条 | 同花色连续三张 |
| 刻子 | 五筒 五筒 五筒 | 三张完全相同 |
| 刻子 | 红中 红中 红中 | 三张红中（作为刻子，非万能牌用途） |

> **注意**：顺子只能由万/条/筒组成，红中不能与字牌组成顺子（本规则无东南西北风）。

### 4.3 红中万能牌（癞子）规则

红中是**万能牌**，可以替代任意一张牌来完成胡牌，规则如下：

#### 4.3.1 替代规则

| 场景 | 红中的作用 | 示例 |
|------|-----------|------|
| 替代顺子中的任意一张 | 红中 = 缺失的那张 | 手中有「二万 三万 红中」→ 视为「一万 二万 三万」或「二万 三万 四万」 |
| 替代刻子中的任意一张 | 红中 = 缺失的那张 | 手中有「五筒 五筒 红中」→ 视为「五筒 五筒 五筒」 |
| 替代将中的一张 | 红中 = 将的配对 | 手中有「七条 红中」→ 视为「七条 七条」 |
| 红中本身作红中 | 三张/四张红中可作为刻子 | 手中有三张红中 → 视为「红中刻子」 |

#### 4.3.2 癞子使用限制

1. **红中不可单独作为将对**：即手中仅 1 张红中 + 1 张其他牌，不能视为一对将。
   - 正确：「七条 + 红中」→ 视为「七条 七条」✓
   - 错误：「红中 + 红中」→ 虽然是两张红中，但可以作为任意将对 ✓（两张红中本身也可以组成任意将）
   
2. **红中替代优先级**：当红中可以替代多种牌时，系统应枚举所有可能的替代方案，只要有一种方案能组成合法胡牌即算胡牌。

3. **红中组刻子**：手中 3 张红中可以不替代任何牌，直接作为"红中刻子"。

#### 4.3.3 胡牌检测算法（伪代码）

```typescript
function canWin(hand: number[], hongZhongCount: number): boolean {
  // hand: 去除已知红中后的手牌
  // hongZhongCount: 手中红中数量
  
  // 尝试每种可能的将
  for (each possible pair in hand + hongZhong_wildcards) {
    remaining = hand - pair;
    wildLeft = hongZhongCount - wildcards_used_for_pair;
    
    // 尝试将剩余牌分解为 4 个面子（用 wildLeft 张红中辅助）
    if (canFormMelds(remaining, 4, wildLeft)) {
      return true;
    }
  }
  return false;
}

function canFormMelds(tiles: number[], meldsNeeded: number, wilds: number): boolean {
  if (meldsNeeded === 0 && wilds === 0) return true;
  if (meldsNeeded === 0 && wilds === 3) return true; // 3张红中组成刻子
  if (tiles.length + wilds < meldsNeeded * 3) return false;
  
  // 尝试提取一个刻子
  for (each triplet candidate) {
    wildsUsed = 3 - count;
    if (wildsUsed <= wilds) {
      if (canFormMelds(remaining, meldsNeeded - 1, wilds - wildsUsed)) return true;
    }
  }
  
  // 尝试提取一个顺子
  for (each sequence candidate in same suit) {
    wildsUsed = 3 - matched;
    if (wildsUsed <= wilds) {
      if (canFormMelds(remaining, meldsNeeded - 1, wilds - wildsUsed)) return true;
    }
  }
  
  return false;
}
```

### 4.4 特殊胡牌牌型（番型）

| 番型 | 说明 | 番数 |
|------|------|------|
| 平胡 | 普通 4 面子 + 1 将 | 1 番 |
| 碰碰胡 | 4 个刻子 + 1 将（无顺子） | 2 番 |
| 清一色 | 所有牌为同一花色（红中除外） | 6 番 |
| 混一色 | 所有牌为同一花色 + 红中 | 2 番 |
| 全求人 | 所有面子均为明面子（碰/杠），最后点炮胡 | 2 番 |
| 杠上开花 | 杠后补摸的牌正好胡牌 | 2 番 |
| 海底捞月 | 摸到牌墙最后一张牌胡牌 | 2 番 |
| 抢杠胡 | 别人明杠时，杠的那张牌正好是自己要胡的牌 | 2 番 |
| 天胡 | 庄家起手 14 张直接胡牌 | 8 番 |
| 地胡 | 闲家第一次摸牌即胡牌 | 8 番 |

> 番型可叠加，总番数 = 各番型之和。

## 5. 特殊规则

### 5.1 自摸

- **定义**：玩家从牌墙摸到的牌使手牌满足胡牌条件
- **行为**：系统弹出"自摸"按钮，玩家可选择胡牌或继续打牌（过）
- **结算**：自摸胡牌，其余三家各付 1 份底分 × 番数 × **自摸倍率（×3）**

### 5.2 点炮（放炮）

- **定义**：玩家打出一张牌后，其他玩家可利用该牌完成胡牌
- **行为**：系统弹出"胡"按钮给可胡牌的玩家
- **结算**：放炮者支付底分 × 番数 × **点炮倍率（×1）** 给胡牌者
- **多人可胡**：按座次优先级，离放炮者最近的玩家优先

### 5.3 碰

- **条件**：任意玩家打出的牌，与自己手中两张相同
- **响应**：弹出"碰"按钮，有 10 秒响应时间
- **效果**：
  1. 将打出的牌与手中两张组成"明刻"，放在桌面
  2. 手牌减少 2 张，需立即打出 1 张
  3. 碰牌后跳过其他玩家的摸牌，直接到碰牌者出牌

### 5.4 杠

#### 5.4.1 明杠

- **条件**：手中有三张相同的牌，其他人打出第四张
- **效果**：四张全部亮出，从牌墙末尾补摸 1 张，再出 1 张

#### 5.4.2 暗杠

- **条件**：手中有四张相同的牌
- **效果**：四张暗扣放在桌面，从牌墙末尾补摸 1 张，再出 1 张
- **时机**：轮到自己摸牌时可选择暗杠

#### 5.4.3 补杠（加杠）

- **条件**：已有明刻（碰过的三张），手中再摸到第四张
- **效果**：将第四张加到明刻上，从牌墙末尾补摸 1 张，再出 1 张

### 5.5 红中特殊规则

1. **红中碰/杠**：
   - 手中 2 张红中 + 别人打出 1 张红中 → 可碰
   - 手中 3 张红中 + 别人打出 1 张红中 → 可明杠
   - 手中 4 张红中 → 可暗杠
   - **注意**：碰/杠红中后，红中不再作为万能牌使用，而是作为普通牌组成刻子/杠子

2. **红中的双重身份决策**：
   - 当手中有 2 张红中时，遇到别人打出第三张红中，玩家可选择"碰"或"不碰"
   - 碰了：红中作为刻子（3 番基础），但失去万能牌灵活性
   - 不碰：红中继续作为万能牌，等待更好的胡牌机会
   - 系统应始终展示两个按钮供玩家选择

3. **红中优先级**：
   - 红中在手牌中始终显示在最左侧，用特殊边框标识
   - 出牌时，如果玩家点击红中，弹出二次确认："确认打出红中？红中是万能牌，打出后不再保留。"

## 6. 计分规则

### 6.1 基础参数

| 参数 | 值 | 说明 |
|------|------|------|
| 底分 | 10 分 | 每局基础分值，可在房间设置中调整 |
| 自摸倍率 | ×3 | 自摸胡牌时的计分倍率 |
| 点炮倍率 | ×1 | 点炮胡牌时的计分倍率 |
| 杠分（明杠） | +10 分 | 每次明杠，其余三家各付 10 分 |
| 杠分（暗杠） | +20 分 | 每次暗杠，其余三家各付 20 分 |
| 杠分（补杠） | +10 分 | 每次补杠，其余三家各付 10 分 |

### 6.2 胡牌计分公式

#### 自摸胡牌

```
单家支付 = 底分 × 番数 × 自摸倍率(3)
胡牌者收入 = 单家支付 × 3（三家各付）
```

**示例**：底分 10，番数 2（碰碰胡），自摸  
→ 每家付 10 × 2 × 3 = 60 分  
→ 胡牌者共得 60 × 3 = **180 分**

#### 点炮胡牌

```
放炮者支付 = 底分 × 番数 × 点炮倍率(1)
胡牌者收入 = 放炮者支付
```

**示例**：底分 10，番数 4（清一色），点炮  
→ 放炮者付 10 × 4 × 1 = **40 分**

### 6.3 杠分结算

杠分在杠发生时**立即结算**，不等到胡牌时：

```
明杠：其余三家各付 10 分 → 杠者得 30 分
暗杠：其余三家各付 20 分 → 杠者得 60 分
补杠：其余三家各付 10 分 → 杠者得 30 分
```

### 6.4 流局结算

```
所有玩家分数不变（杠分已结算的不退还）
庄家轮转到下一位
```

### 6.5 完整结算示例

```
底分 = 10
玩家 A（庄家）：碰了 1 次，暗杠 1 次
玩家 B：明杠 1 次
玩家 C：自摸胡牌，清一色(4番) + 碰碰胡(2番) = 6番

杠分结算：
- A 暗杠：B、C、D 各付 20 → A +60
- B 明杠：A、C、D 各付 10 → B +30

胡牌结算（自摸，6番）：
- 每家付 10 × 6 × 3 = 180
- C 收入 = 180 × 3 = 540

最终变动：
- A: +60 - 180 = -120
- B: +30 - 180 = -150
- C: +540
- D: -180 - 20 - 10 = -210 (还需减去杠分支付)
```

> **开发注意**：杠分和胡牌分分开计算，各自独立转账，避免混淆。

## 7. AI 难度设计

### 7.1 总体设计思路

AI 采用**打分评估 + 概率调整**的策略框架。每个决策点（出牌、碰、杠、胡）都通过评估函数计算各选项的得分，再根据难度等级引入不同幅度的随机偏差。

### 7.2 简单 AI（Easy）

| 维度 | 策略 |
|------|------|
| 出牌 | 随机从"可出牌"中选择，60% 概率打孤张（无关联的牌），40% 随机出 |
| 听牌判断 | 不做听牌计算，只在系统提示胡牌时才胡 |
| 碰/杠 | 50% 概率选择碰，30% 概率选择杠 |
| 红中使用 | 倾向于早期打出红中（不理解万能牌价值） |
| 防守 | 完全不防守，不读牌 |

**实现要点**：
```typescript
function easyAIDiscard(hand: number[]): number {
  // 60% 打孤张，40% 随机
  if (Math.random() < 0.6) {
    const isolated = findIsolatedTiles(hand); // 与其他牌无关联的牌
    if (isolated.length > 0) return randomPick(isolated);
  }
  return randomPick(hand);
}
```

### 7.3 中等 AI（Medium）

| 维度 | 策略 |
|------|------|
| 出牌 | 优先打孤张和边张，保留中张和连搭牌 |
| 听牌判断 | 实时计算听牌数，选择进张最多的打法 |
| 碰/杠 | 碰牌时评估是否有利于向听，有利则碰（80%），否则过 |
| 红中使用 | 保留红中到中后期，优先用于组成面子 |
| 防守 | 轻度防守：当其他玩家明显听牌时，避免打生张 |
| 效率 | 贪心算法：每次出牌选择"向听数减少最多"的方案 |

**核心评估函数**：
```typescript
function evaluateDiscard(hand: number[], candidate: number): number {
  const remaining = removeTile(hand, candidate);
  const shanten = calcShanten(remaining); // 计算向听数
  const waits = countWaits(remaining);     // 计算进张数
  
  // 综合评分 = 进张数权重 × 进张数 - 向听数惩罚
  return waits * 10 - shanten * 50;
}
```

### 7.4 困难 AI（Hard）

| 维度 | 策略 |
|------|------|
| 出牌 | 全局最优搜索：考虑牌池信息，选择安全度 + 效率最高的牌 |
| 听牌判断 | 精确计算所有可能的胡牌组合，选择收益最大的听牌方案 |
| 碰/杠 | 碰牌时精确计算碰后向听数变化，只碰净收益为正的情况 |
| 红中使用 | 深度评估红中替代方案，穷举所有可能的分配，选择胡牌概率最高的 |
| 防守 | 积极防守：读牌池、推算其他玩家手牌，打出安全度最高的牌 |
| 信息利用 | 记录所有已打出的牌和碰/杠信息，推算剩余牌分布 |
| 多目标优化 | 平衡进攻（尽快胡牌）和防守（避免点炮） |

**读牌策略**：
```typescript
function calcTileSafety(tile: number, discardPool: Set<number>): number {
  const totalInGame = 4; // 每张牌最多 4 张
  const discarded = countDiscarded(tile, discardPool);
  const remaining = totalInGame - discarded;
  
  // 已打出越多 → 越安全（别人手中越不可能有这张）
  // 已打出 3 张 → 极安全
  // 已打出 0 张 → 危险
  return discarded / totalInGame; // 0~1，越大越安全
}
```
