# 红中麻将 v1.0 — Bug清单

> 审查人：项目经理

## 🔴 严重Bug（必须修复）

### BUG-001：出现"0万"不存在的牌
- **现象**：手牌中出现tile ID=0，显示为"0万"
- **原因**：牌ID编码逻辑有误。万子应该是1-9万，但ID=0被当成了"0万"
- **期望**：万子ID范围应为1-9，不存在0万
- **修复要求**：检查buildDeck()和tileNumber()，确保万子从1开始

### BUG-002：风位标注错误
- **现象**：显示"你(西)"、"东(AI)"、"北(AI)"、"南(AI)"
- **原因**：drawPlayerLabels()中风位映射错误
- **期望**：应该是"你(南)"，对面"北(AI)"，左边"东(AI)"，右边"西(AI)"
- **修复要求**：修正风位映射，玩家永远是南家

### BUG-003：计分系统未实现
- **现象**：分数一直是[0,0,0,0]，胡牌后分数不变
- **原因**：endGame()中没有计算和更新分数
- **期望**：自摸×3三家赔，点炮×1放炮者赔，杠分明杠+10/暗杠+20
- **修复要求**：实现完整计分逻辑，胡牌后显示分数变化

## 🟡 中等Bug（应该修复）

### BUG-004：游戏开始时有牌被选中
- **现象**：开局第4张牌上移，像是被选中了
- **原因**：selectedIdx初始值可能不是-1，或渲染逻辑有误
- **期望**：开局没有任何牌被选中
- **修复要求**：确保开局selectedIdx=-1

### BUG-005：红中打出无二次确认
- **现象**：点击红中直接打出，没有确认弹窗
- **原因**：未实现
- **期望**：策划文档要求"打出红中需二次确认"
- **修复要求**：点击红中时弹出确认框"确认打出红中？红中是万能牌，打出后不再保留。"

### BUG-006：AI左右手牌方向错误
- **现象**：左右AI的牌是横向排列的，但显示方向不对
- **原因**：Canvas旋转角度可能有误
- **期望**：左侧AI牌竖向排列（90°旋转），右侧AI牌竖向排列（-90°旋转）
- **修复要求**：修正左右AI牌的旋转渲染

---

## 🟢 低优先级（建议修复）

### BUG-007：没有新局按钮（游戏中）
- **现象**：游戏进行中没有"新局"按钮，只有结果弹窗里有
- **修复要求**：在状态栏附近加一个"新局"按钮

### BUG-008：没有难度选择
- **现象**：默认hard难度，无法选择
- **修复要求**：开局前显示难度选择（简单/中等/困难）

### BUG-009：AI暗杠概率只有50%
- **现象**：AI有暗杠时只有50%概率执行
- **原因**：代码中 `Math.random() > 0.5`
- **期望**：暗杠是好牌，应该100%执行
- **修复要求**：改为100%执行暗杠
