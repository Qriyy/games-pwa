# 红中麻将 — 美术设计文档

> 版本：1.0  
> 项目：红中麻将（网页PWA游戏）  
> 工作室：棋牌乐园游戏工作室 · 美术部  
> 技术约束：单HTML + Canvas/DOM + Web Audio，无外部资源依赖

---

## 1. 整体视觉风格

**风格定位：** 暗色中式古典，庄重大气，带有木质感和丝绒质感。

- **桌面背景**：深色仿红木纹理，用CSS径向渐变模拟灯光聚焦效果
- **氛围**：如同深夜灯光下的牌桌，温馨而专注
- **装饰元素**：金色描边、暗纹回字纹、云纹点缀
- **阴影风格**：多层投影营造立体感，牌面有轻微内阴影
- **圆角**：牌面圆角 `4px`，按钮圆角 `8px`，面板圆角 `12px`

## 3. 牌面设计

### 3.1 牌面尺寸

| 场景 | 宽 | 高 | 单位 |
|------|----|----|------|
| 手牌（自己） | 48 | 68 | px（@1x基准） |
| 手牌（对手·横置） | 32 | 44 | px |
| 弃牌区 | 28 | 40 | px |
| 副露牌 | 36 | 52 | px |
| 摸牌（自己） | 52 | 74 | px（略大突出） |

> 实际渲染按 `baseSize × viewportScale` 缩放

### 3.2 牌面视觉规范

**底牌样式：**
- 底色：`#F5E6C8`（暖米白）
- 边框：`1px solid #C8B088`，底部 `2px solid #A89068` 模拟牌厚度
- 圆角：`3px`
- 内阴影：`inset 0 1px 0 rgba(255,255,255,0.3)` 模拟高光
- 外阴影：`2px 2px 4px rgba(0,0,0,0.4)`

**牌背样式（对手手牌）：**
- 底色：`#2A5A3A`
- 中央图案：CSS绘制的回字纹 `#3A7A4A`
- 边框：`1px solid #1A3A28`

### 3.3 数牌设计

#### 万子（1万~9万）
- 底部：汉字数字（壹贰叁…玖），字体大小 `16px`
- 顶部：小号「万」字，字体大小 `12px`
- 颜色：`#1A1A1A`（纯黑偏深灰）

#### 条子（1条~9条）
- 用竖向绿色竹节条表示，1条为特殊带花纹的竹节
- 颜色：`#1A6B3C`（翠绿）
- 竹节宽度：`4px`，间距 `2px`

#### 筒子（1筒~9筒）
- 用圆形铜钱图案表示
- 颜色：`#1A5A8A`（靛蓝）
- 圆形直径：`12px`，内部十字或空心装饰

### 3.4 红中的特殊样式 ⭐

红中是本游戏的核心牌，需有显著视觉区分：

- **字体**：大号「中」字，字体大小 `28px`（牌面内最大）
- **颜色**：`#C43B2A`（正红）
- **字重**：bold / 700
- **描边**：`1px #8A1A0A` 外描边增强辨识度
- **背景微调**：牌面底色 `#FFF0E0`（比普通牌偏暖偏亮）
- **选中态**：上移 `12px`，底部加 `0 4px 12px rgba(196,59,42,0.5)` 红色光晕
- **摸到红中时**：触发一次短暂的 `#E85545` 脉冲闪光（300ms）

## 5. 字体方案

### 5.1 中文字体

```css
font-family: "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

- 优先级：苹方 → 微软雅黑 → 思源黑体 → 系统无衬线
- 牌面汉字（万/条/筒/风牌文字）：
  ```css
  font-family: "KaiTi", "STKaiti", "SimSun", serif;
  ```
  使用楷体/宋体渲染，更有传统书法韵味

### 5.2 数字字体

- 积分/倒计时等 UI 数字：
  ```css
  font-family: "DIN Alternate", "Helvetica Neue", "Arial", sans-serif;
  ```
- 备选（无外部资源时用系统等宽字体）：
  ```css
  font-family: "Consolas", "Courier New", monospace;
  ```

## 6. 动效设计

### 6.1 摸牌动效

**触发：** 轮到自己摸牌时

- 牌从牌堆中央飞向手牌区右侧
- 路径：弧线（贝塞尔曲线），时长 `400ms`
- 缓动：`ease-out`
- 牌飞行中略微旋转（`rotate: -10deg → 0deg`）
- 到达手牌后：
  - 牌背 → 翻转为正面（Y轴 `rotateY: 180deg → 0`，`200ms`）
  - 轻微弹跳效果（`scale: 1 → 1.1 → 1`，`150ms`）
- 若摸到红中：额外脉冲红色光晕 `300ms`

### 6.2 出牌动效

**触发：** 玩家打出一张牌

- 牌从手牌区滑向弃牌区对应位置
- 路径：直线，时长 `250ms`
- 缓动：`ease-in-out`
- 到达弃牌区后：轻微落地震动效果（`translateY: -4px → 0`，`100ms`）
- 金色发光边框闪烁 `1.5s` 后淡出

### 6.3 胡牌庆祝特效

**触发：** 有玩家胡牌

- 全屏遮罩淡入（`rgba(0,0,0,0.6)`，`300ms`）
- 中央大字「胡」从中心弹出（`scale: 0 → 1.2 → 1`，`500ms`，`ease-out-back`）
- 「胡」字颜色：`#C43B2A`，描边 `#D4A545` 金色
- 周围粒子效果：金色小圆点向外扩散（Canvas粒子，50-80个，`1.5s`）
- 底部显示胡牌详情（牌型、番数），`800ms` 延迟后淡入
- 整体持续 `3s` 后自动关闭

### 6.4 碰/杠提示特效

**触发：** 可碰/可杠时

- 操作按钮从底部弹入（`translateY: 100% → 0`，`300ms`，`ease-out`）
- 按钮有持续轻微脉冲（`scale: 1 ↔ 1.03`，`1s` 循环）吸引注意
- 碰：相关牌在弃牌区发光 `#D4A545` 金边
- 杠：相关牌发光更强，边框 `3px`，带 `box-shadow` 扩散

### 6.5 倒计时进度

- 圆形进度环，颜色从 `#D4A545`（金）渐变到 `#C43B2A`（红）
- 最后5秒：进度环闪烁 + 文字跳动

## 7. 音效设计

> 所有音效使用 Web Audio API 合成，不依赖外部音频文件

### 7.1 摸牌音

- **风格：** 清脆木质敲击
- **合成方式：** 短促噪声脉冲（`duration: 80ms`）+ 高频谐振（`800-1200Hz`）
- **音色：** 像手指轻叩桌面木牌的声音
- **音量：** `0.3`

### 7.2 出牌音

- **风格：** 稍重的牌落桌面声
- **合成方式：** 中频噪声脉冲（`duration: 120ms`）+ 低频共振（`200-400Hz`）
- **音色：** 厚实的「啪」一声
- **音量：** `0.4`

### 7.3 碰/杠音

- **风格：** 短促有力的双击/三击
- **碰：** 两声快速敲击，间隔 `60ms`
- **杠：** 三声快速敲击，间隔 `50ms`，最后一声略重
- **合成方式：** 类似出牌音，叠加更多中低频
- **音量：** `0.5`

### 7.4 胡牌音

- **风格：** 欢快的锣鼓点 + 上升音阶
- **合成方式：**
  1. 开头：低频大鼓声（`100Hz`，`200ms`，`0.6`）
  2. 中段：快速上升音阶（`C4→E4→G4→C5`，每音 `100ms`）
  3. 结尾：高频余韵（`2000Hz`，渐弱 `500ms`）
- **音量：** `0.7`

### 7.5 其他音效

| 事件 | 描述 | 时长 | 音量 |
|------|------|------|------|
| 按钮点击 | 轻微「嗒」声 | 50ms | 0.2 |
| 倒计时最后5秒 | 每秒一声低频提示 | 80ms | 0.3 |
| 流局 | 低沉「嗡」声 | 500ms | 0.4 |
| 错误操作 | 短促双音（降调） | 150ms | 0.3 |

## 8. 响应式方案

### 8.1 断点定义

| 断点 | 宽度范围 | 设备类型 |
|------|----------|----------|
| `sm` | `< 640px` | 手机竖屏 |
| `md` | `640px – 1024px` | 手机横屏 / 小平板 |
| `lg` | `> 1024px` | 电脑 / 大平板 |

### 8.2 手机竖屏（sm）

- **方向：** 纵向布局，画面从上到下排列
- **牌面缩放：** 基准尺寸 × 0.6
- **手牌：** 底部横向排列，可左右滑动查看超出屏幕的手牌
- **对手手牌：** 顶部显示对面（缩小），左右对手隐藏，仅显示副露
- **弃牌区：** 居中，4列压缩为紧凑排列
- **操作按钮：** 底部固定，横排一排
- **触摸优化：** 手牌触摸上移 `16px` 为选中态，再点打出
- **最小触摸目标：** `44×44px`

### 8.3 手机横屏 / 小平板（md）

- **方向：** 横向布局，接近电脑布局
- **牌面缩放：** 基准尺寸 × 0.8
- **四方向对手：** 全部显示
- **弃牌区：** 标准4列布局
- **操作按钮：** 右下角纵向排列或底部横向

### 8.4 电脑横屏（lg）

- **方向：** 标准桌面布局（见第4节）
- **牌面缩放：** 基准尺寸 × 1.0
- **鼠标交互：** hover 手牌上移 `8px`，点击选中上移 `12px`
- **弃牌区：** 充分利用空间
- **操作按钮：** 底部居中横排

### 8.5 Canvas自适应策略

```
设计基准：1280 × 720 (16:9)
渲染时：
  scale = min(canvasWidth / 1280, canvasHeight / 720)
  所有坐标和尺寸 × scale
```

- 保持宽高比，必要时黑边填充（`letterbox`）
- DOM元素使用 `vw/vh` + `clamp()` 自适应字号

## 9. PWA图标设计

### 9.1 图标描述

- **形状：** 圆角正方形，圆角 `22%`（iOS风格）
- **背景：** 从 `#1B3A2D` 到 `#0F2218` 径向渐变
- **中心图案：** 一张红中麻将牌的正面特写
  - 牌面占图标 60% 面积
  - 底色 `#F5E6C8`，圆角 `6px`（按比例）
  - 中央大号「中」字，`#C43B2A`，加粗
  - 牌面有轻微 `rotateY: -15deg` 透视，增加立体感
- **装饰：** 牌面下方有 `#D4A545` 金色光芒线条扩散
- **投影：** 牌面底部 `0 4px 8px rgba(0,0,0,0.5)`

### 9.2 图标尺寸

| 用途 | 尺寸 |
|------|------|
| favicon | 32×32, 16×16 |
| PWA icon (Android) | 192×192, 512×512 |
| PWA icon (iOS) | 180×180 |
| Apple touch icon | 180×180 |
| Windows tile | 150×150 |

### 9.3 生成方式

使用 Canvas API 在运行时绘制图标（不依赖外部图片资源）：

```javascript
// 伪代码示意
function drawIcon(size) {
  ctx.fillStyle = radialGradient('#1B3A2D', '#0F2218')
  ctx.roundRect(0, 0, size, size, size * 0.22)
  ctx.fill()
  
  // 牌面
  ctx.save()
  ctx.translate(size*0.5, size*0.52)
  ctx.rotateY(-0.26) // 约-15度
  ctx.fillStyle = '#F5E6C8'
  ctx.roundRect(-size*0.22, -size*0.3, size*0.44, size*0.6, size*0.04)
  ctx.fill()
  
  // 「中」字
  ctx.fillStyle = '#C43B2A'
  ctx.font = `bold ${size*0.3}px KaiTi, serif`
  ctx.fillText('中', 0, size*0.08)
  ctx.restore()
  
  // 金色光芒
  drawGoldenRays(size*0.5, size*0.75, size*0.15)
}
```
